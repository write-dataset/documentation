WRITE Digital Archive


The WRITE dataset documentation

WRITE digital archive aims to store, preserve, analyse and disseminate the data collected through the WRITE project.

The WRITE project - New Forms of Calligraphy in China: A Contemporary Culture Mirror - wants to investigate how the emergence of new forms of calligraphy in contemporary China has been playing a pivotal role in the re-definition of Chinese cultural identity. Given the evergrowing presence of calligraphy in contemporary arts, the central core of the project is to systematically analyse all these new art forms.

This dataset documentation stores detailed information about the data collected by the domain experts involved in the WRITE project.




The core of the WRITE dataset are four collections corresponding to four types of art:


Visual Arts
Decorative and Applied Arts
Graffiti Art
Performing Arts

Starting from these art types, by employing WRITE data model, this dataset structures and preserves all the data that represent the miscellaneous nature of the Chinese contemporary calligraphic phenomenon. The choice to employ the Linked Open Data and FAIR principles was made in order to bridge the data silos and to be introduced into the Knowledge Graph.

Using LOD will also contribute to the analysis of the dataset on three different levels: 1) artistic, 2) linguistic and 3) socio-political-economic. Artistic analysis deals with formal and stilistic aspects of the artworks, linguistic analysis comprises both the presence of calligraphy within the artworks as well as the critical and other texts related to the artworks, while socio-political-economic analysis offers information about clients, patrons, public response, etc.

A special place in this data model is dedicated to the so-called "calli-writing" units, which are the core feature for the calligraphic investigation of this project (see more in the model -> Calli-Writing Unit section).

In this dataset documentation you can get an insight into the particularities of the WRITE data model (see model), see how it is applied in 4 representative examples for each of the four collections (see case studies), and finally, view, browse and download the complete dataset and ontologies that constitute the data model (see RDF data).
